# 改动工作流补充（Tsb ERP 通用）

> 项目特定值（标准页/前缀）→ 先读 [../t-page/profiles.md](../t-page/profiles.md)。

## 必问单选题总表

**统一规则**（全 Skill 共用，无例外）：

1. 用 **AskQuestion** 工具发起单选（每题 2~4 项，选项具体可执行）
2. **选定前**：不写代码 · 不 Grep 删改 · 不大范围动文件
3. 一题一事；多件事拆多题依次问
4. 禁止开放式糊问 · 禁止心里默认（含默认主单、默认 pageId、静默不抽离）

**题目清单**（按场景）：

1. **任务类型** — 改代码前类型不明 →「Bug 还是新需求？」→ A Bug · B 新需求/改动
2. **主单/详情范围** — 页面范围不清；删改功能未指明主单/详情 →「改哪个页面？」→ A 主单 · B 详情 · C 两者（换对应文件夹名）
3. **同页多入口** — 范围已定，同页多个按钮/批量入口 →「改哪些入口？」→ A 入口1 · B 入口2 · C 都改
4. **i18n 是否动** — 可能涉及文案 →「是否改 locales？」→ A 不改复用 key · B 新增 key（接 #5）；**参考他模块时必查 key 文案是否指当前业务**
5. **i18n 新增策略** — 确定要新增 key →「zh-CN 与 en 怎么写？」→ A 两文件都中文 · B 中英文区分
6. **pageId** — 新增 OP_HEADERS / 新页操作日志 →「pageId 怎么处理？」→ A 已提供数字 · B 后续处理 `"需要后台配置！"`
7. **公共代码抽离** — 多文件可复用、大改 →「要不要抽离？」→ A 不抽离照抄 · B 允许抽离（说明范围）
8. **Bug 兄弟页** — 同模块多页可能有同样问题 →「是否同步改兄弟页？」→ A 仅当前页 · B 兄弟页一并改
9. **新模块详情** — 新建模块未说要不要详情 →「是否做详情页？」→ A 仅列表 · B 列表+详情（th 项目注意：详情多内嵌 Detail.vue）
10. **权限码** — 新增 AUTH_CODES →「权限码怎么处理？」→ A 已提供后端码 · B 后续处理 `"需要后台配置！"`
11. **弹窗选型** — Drawer/Dialog 均可、未指定 →「用哪种容器？」→ A Drawer · B Dialog · C 照抄标准页同功能
12. **B 档大改** — registry B 档页要大改结构 →「对齐标准到什么程度？」→ A 仅修当前问题 · B 对齐同模块 A 档
13. **字段设置** — 弹窗/抽屉/详情表单/grid **新增字段** →「接入哪些维度？」→ A~G 可多选
14. **Tab/弹窗可编辑** — Tab 或弹窗**新增改数据/确认按钮** → 是否支持保存？→ A 可编辑 · B 只读/预览 · C 照抄同 Tab 相邻按钮 · D 指定哪些 Tab 可保存

细则 → [t-field](../t-field/SKILL.md) · [t-tab](../t-tab/SKILL.md)

---

Bug：报错、数据不对、按钮无效
新需求：新页面、新按钮、新字段
需求改动：在旧页上加/改功能
优化/重构：先问用户归类

Bug 常见漏改：列表忘了详情 · columns 忘了 type · 只改 zh 忘了 en · 改了 hook 没查调用方

## 对照源码

- Bug 修复：当前项目标准页同逻辑位置（见 [../t-page/anchors.md](../t-page/anchors.md)）
- 新需求列表：复制当前项目 `${标准页}` 改命名
- 新需求详情：复制当前项目详情模板（ts→`${标准页}Detail`；th→内嵌 Detail.vue 或 `${标准页}Detail`）

新需求必做：API 三件套 · OP_HEADERS（列表 + 详情，见 t-oplog）· 路由 · auth · i18n · virtualYConfig · useColWidthSave

i18n：非必须不改 locales；**新增 key 前必问**；**参考他模块必查 locales 文案**，禁止错模块提示
pageId：**新增 OP_HEADERS 前必问**；选后续处理则写 `"需要后台配置！"`
主单/详情：说主单只改主单 · 说详情只改详情 · **删改功能名未指明页面必问** · 范围模糊必问单选

全模块路由/Tab 见 t-registry · 文件命名见 [../t-page/naming.md](../t-page/naming.md)

## 批量任务与抽离（禁脚本红线）

### 禁脚本（绝对红线）

- ❌ **禁止写任何脚本处理 ERP 代码**：sed/awk/grep 批量替换、.ps1/.sh/.mjs/.py 批处理——一律禁
- ❌ 哪怕"就这一次省事"也不行；哪怕任务涉及几十个文件也不行

### 批量任务怎么处理（按模块拆 subagent 并行）

跨多文件/多模块的同类任务，**按业务模块拆解**，AI 逐文件读改：

1. 主 Agent 扫描大菜单（`ls src/views/`）确定涉及哪些模块
2. **每个涉及模块派一个 `t-mod` 实例**并行（采购实例/销售实例/跟单实例…）
3. 每个实例**锁定自己模块目录**，用 Read/Edit/Write **逐文件读着改**
4. 实例间不通信、不越界；遇共享组件被多模块用 → 报告主 Agent 裁决
5. 收齐后统一审查（spec-reviewer + impact-surveyor）

详见 [../t-page/orchestration.md#模块并行拆解](../t-page/orchestration.md#模块并行拆解批量任务--全模块任务--同类多模块任务)

### 单模块内多文件

不跨模块、只是同模块多文件：
- 仍**逐文件 AI 改**（Read 一个 Edit 一个），不写脚本
- 可复用写法在各文件内照抄

### 抽离公共代码

- 大改且多文件可复用时，**必须 AskQuestion**（见总表 #7）
- 未问不得抽离；用户选 A → 各文件独立照抄
- 用户选 B → 才可抽离，且不能影响现有功能

## 最简代码

- 只改问题所需，最少行数
- 照抄标准页一小段，不新造 hook/工具层
- 不加以防万一的分支、不顺手重构
- 冲突时以更简单为准

## 疑问与歧义

以下情况必须先单选题，选定后再继续，禁止猜测：

- 意图不清 · 需求多义 · 方案多条路径
- **主单 vs 详情范围不清**（如只说「客户信息那个」「采购订货页面」）

优先 AskQuestion，每题 2~4 选项；选定前不写代码。

- C. 列表 + 详情都改

**方案分叉 — 是否抽公共代码**

- A. 不抽离，各文件内照抄（默认）
- B. 允许抽离，范围限于 xxx hook

**Bug 范围 — 是否同步兄弟页**

- A. 仅改当前页
- B. 同模块兄弟页同位置一并改

**i18n — 是否动文案**

- A. 不改 locales，复用已有 key
- B. 新增 key（选定后另问「两文件都中文 / 中英文区分」）

**i18n — 新增 key 时 zh-CN 与 en 怎么写（必问）**

- A. 两文件都用中文（zh-CN + en 新增项均为中文）
- B. 中英文区分（zh-CN 中文，en 英文翻译）

## 参考他模块时的 i18n

**触发**：照抄他模块组件/逻辑，沿用或拟用 `t('xxx')`。

**必做**：

```bash
# 查 key 实际文案（示例 key 替换为代码里的；命名空间按当前项目 profiles locales 列）
rg "^  m\.m123:" ${locales目录}/zh-CN.yaml ${locales目录}/en.yaml
```

1. 打开 locales，读该 key 的**完整中文**
2. 判断是否**当前模块、当前功能**专用文案
3. 指错模块 / 指错业务 → **新增 key**，按当前场景重写，禁止复用

**反例**：

| 当前在做 | 错复用的 key 文案 | 应做 |
|----------|-------------------|------|
| 客户资料 · 附件上传 | 「跟单日志附件最多上传 9 个」 | 新建 key，如「客户资料附件最多上传 9 个」 |
| 采购订货 · 保存失败 | 「销售订单已关闭，无法修改」 | 新建 key，改为采购语境 |

**可复用**：真正通用、无业务名的 key（如 `buttons.btn14` 确认、`g.g640` 无权限）。

**禁止**：只抄 `t('m.mxxx')` 不查 yaml；改已有 key 翻译（会波及原模块）。

**pageId — 新增 OP_HEADERS 时（必问）**

- A. 我已提供 pageId（按给定数字写入各操作项）
- B. 后续处理（`pageId` 统一写 `"需要后台配置！"`，待后台补全）

**主单 vs 详情 — 范围模糊时（必问）**

通用示例：

- A. 只改主单 `${views前缀}Xxx`
- B. 只改详情（明细页）
- C. 主单 + 详情两个页面都改

用户已明确说「主单」→ 只改 A，不问；已明确说「详情/明细」→ 只改 B，不问。

**删 / 改功能 — 只提模块 + 功能名（必问，第一题）**

- A. 只改主单（单行/批量入口）
- B. 只改详情（按明细）
- C. 主单 + 详情都删

**同页多入口 — 范围已定后再问（第二题，按需）**

主单内同一功能可能单行与批量两入口：

- A. 只删单行入口
- B. 只删批量入口
- C. 两个都删

**共享组件 — 蓝图「不改」须列出**

删主单弹窗时，若该组件仍被兄弟模块引用 → 禁止删组件文件，只删当前主单入口。

**任务类型 — 改代码前（必问，用户已说明则跳过）**

- A. Bug 修复
- B. 新需求 / 需求改动

**新模块 — 是否做详情（必问，未说明时）**

- A. 仅列表页
- B. 列表 + 详情（ts 用 `${views前缀}XxxDetail` 目录；th 视情况内嵌或独立）

**权限码 — 新增 AUTH_CODES（必问）**

- A. 我已提供后端权限码（按给定写入 auth.ts）
- B. 后续处理（值写 `"需要后台配置！"`，待后台补全）

**弹窗选型 — Drawer / Dialog 均可时（必问）**

- A. `el-drawer` + Drawer Header（复杂表单，照抄 AEDrawer）
- B. `el-dialog` + modalOptions（简单表单，照抄 AEDialog）
- C. 照抄标准页同功能容器（说明参照哪页）

**B 档大改 — registry 标注 B 档的页要大改（必问）**

- A. 仅修当前问题，不顺手对齐 A 档
- B. 对齐同模块 A 档列表栈写法

**字段设置 — 新增弹窗/抽屉/表单/grid 字段（必问，可多选）**

> 新字段 `xxx` 要接入哪些字段设置？

- A. 编辑可见（`setField` type 4）
- B. 可编辑（`setField` type 2）
- C. 详情可见（`setField` type 1）
- D. 列表可见（`showList` / columns）
- E. 批量编辑可见（type 5）
- F. 批量编辑可编辑（type 3）
- G. 暂不接字段设置（须你确认例外）

默认照抄同页相邻已有字段的组合。细则 → [t-field](../t-field/SKILL.md)

**Tab/弹窗可编辑 — 新增改数据或确认按钮（必问）**

> 当前 Tab/弹窗是否支持编辑保存？新按钮怎么处理？

- A. 可编辑 — 显示确认/保存，接 `isEdit` / `*SaveHidden`
- B. 只读/预览 — 隐藏确认/保存，编辑类 disabled
- C. 照抄同 Tab 相邻按钮（默认）
- D. 指定哪些 Tab id / dialogType 可保存

细则 → [t-tab](../t-tab/SKILL.md)
